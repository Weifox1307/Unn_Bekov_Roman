# Лабораторные работы по дисциплине "Технологии баз данных"
## Создание базы данных "Quotation" в среде PostgreSQL

Данный репозиторий содержит комплект выполненных лабораторных работ (1–6), реализованных в СУБД PostgreSQL с использованием инструмента pgAdmin 4.

### 📋 Состав работ:
*   **00_setup_tables.sql**: Подготовка структуры БД. Создание основных таблиц (`quotations`, `qitem`, `agents`, `admin_log`) и заполнение их тестовыми данными.
*   **Lab_01**: Построение динамического ряда суммы предложений по месяцам с использованием группировки и форматирования даты.
*   **Lab_02**: Создание хранимой процедуры для динамической генерации представления (VIEW) по заданному диапазону дат.
*   **Lab_03**: Реализация триггера `BEFORE INSERT` для валидации вводимых цен (проверка вхождения в интервал между ценой вендора и прайс-листом).
*   **Lab_04**: Реализация триггера `AFTER INSERT` для журналирования (логирования) действий пользователя в таблицу `admin_log`.
*   **Lab_05**: Реализация шаблона разреженных данных (Star-scheme / EAV) для хранения дополнительных атрибутов агентов (на примере количества детей из вордовского файла).
*   **Lab_06**: Использование аналитических функций для расчета процентного вклада каждого агента в общий оборот компании.

### 🛠 Инструкция по запуску:
1. Создаем новую базу данных в pgAdmin 4 (например, `Quotation_DB`).
2. Открываем Query Tool нажатием правой кнопки мыши по названию нашей БД и выполняем файл **`00_setup_tables.sql`**. Это создаст все необходимые таблицы и связи.
3. После этого файлы лабораторных работ (**Lab_01** – **Lab_06**) можно выполнять по отдельности в любом порядке.

### 💡 Как это работает:
* Проверка данных: В 3-й и 4-й лабах настроил триггеры. Это такие скрипты, которые сами срабатывают при добавлении записей: один проверяет, чтобы цена не была "левой", а второй записывает историю действий в лог.
* Удобные отчеты: В 6-й лабе вместо сложных вложенных запросов использовал оконную функцию OVER(). С ней гораздо проще посчитать долю каждого агента в общем обороте одной строкой.
* Хранение доп. данных: В 5-й лабе не стал добавлять лишние колонки в основную таблицу агентов (чтобы не было кучи пустых ячеек), а вынес редкие свойства (вроде количества детей) в отдельную таблицу. Это и есть та самая схема "разреженных данных".
* Единая структура: Специально собрал создание всех таблиц в один файл **`00_setup_tables.sql`**. Это намного удобнее: один раз запускаешь скрипт, и вся база готова под все лабы сразу. Не нужно для каждой задачи заново пересоздавать таблицы или бояться, что они конфликтуют между собой, с чем уже не раз приходилось сталкиваться в ходе решения.
